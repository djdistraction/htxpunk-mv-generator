# 🏗️ System Architecture

## Overview

The VoodooHut MV Generator is an 8-stage pipeline that transforms a raw audio file into a full animated music video. Each stage is an independent async task, with two human approval checkpoints before expensive generation steps.

```
Audio File → [Stage 1] Analysis → [Stage 2] Treatment ──APPROVAL──►
            [Stage 3] Elements → [Stage 4] Backgrounds → [Stage 5] Elements →
            [Stage 6] Storyboard ──APPROVAL──► [Stage 7] Video Clips →
            [Stage 8] Final Assembly → MP4
```

## Data Flow: Continuity Bible

The `BIBLE.md` is regenerated by `export_bible()` at any time and reflects
the current state of all assets. It tracks:

- Visual treatment (approved logline, style, palette)
- All element states with asset URLs and FLUX prompt used
- Background registry with asset URLs
- Storyboard panel order with composition details
- Clip registry with RunwayML task IDs
- Final video URL + production log

## Infrastructure Requirements

**Runtime dependencies (local/server):**
- Python 3.11+
- FFmpeg (in PATH)
- Redis (via Docker or managed service)

**Cloud services (API keys required):**
- OpenAI (Whisper + GPT-4o)
- Replicate (FLUX.1-dev)
- RunwayML (Gen-4)
- Supabase (PostgreSQL)
- Cloudflare R2 (object storage)

**Local Python packages doing heavy lifting:**
- `rembg` — background removal (runs a local ONNX model)
- `Pillow` — compositing (pure Python, no extra deps)
- `pydub` — audio duration extraction

## Scaling Considerations

- **Celery workers** can be scaled horizontally for parallelism
- **Image generation** is the bottleneck — FLUX.1 takes ~15-30s/image
- **Video generation** is the most expensive — RunwayML ~$0.05/clip × 36 clips = ~$1.80
- **Stagger clip generation** using Celery chord to run 4 concurrent clips max
- **R2 zero egress** means no bandwidth cost for downloading assets between stages
